# Changelog

以下仅保留当前版本仍成立的用户侧变更与能力说明。

## 2026-04-30

### 作物与分类资源补齐

- `gameConfig/plant_images/stages/作物` 已补齐缺失作物资源与映射：新增 `红云飞片`、`黄金果`、`卡特兰`、`孔雀草`、`蓝铃花`、`欧石楠`、`石竹花`、`宋梅`、`针垫花` 等缺失作物目录，并把作物映射补齐到 `141` 条，`哈哈南瓜` 也已补入作物映射。
- 作物运行时解析已增加 `crop_level_mapping.json` 回退链路；即便 `Plant.json` 中缺少个别新作物，土地详情与仓库页也能按 `crop_id / seed_id / fruit_id` 命中作物名称、等级与主图。
- `礼包与宝箱` 资源已与分类映射完全对齐，补入 `天工种子包(测试)` 图片与映射，仓库页可直接显示该物品主图。
- 新增独立 `gameConfig/plant_images/stages/黄金果实` 资源目录，已补齐 `18` 条黄金果实映射及对应图片资源；其中既支持普通单图物品，也支持 `黄金·卡特兰`、`黄金·黄金果`、`黄金·哈哈南瓜` 等带阶段子目录的黄金果实资源。
- 分类资源加载逻辑已扩展为同时支持“目录内直接放图片”和“目录内再按物品分阶段子目录存图”两种模式；仓库页现在可从 `黄金果实` 这类子目录资源中自动提取 `00_主图 / 作物图 / 01_种子` 作为物品主图。

### 背包种植防误种

- 背包优先种植不再只硬编码屏蔽 `爱心果`；现已改为按作物配置中的占地尺寸统一识别四格作物。
- 像 `哈哈南瓜`、`爱心果` 这类 `size >= 2` 的四格作物种子，现在会在背包种子优先级列表中显示为 `不可种植`，自动种植执行时也会被直接跳过，避免误种植导致异常。

## 2026-04-29

### 自动施肥、枯萎收取与日志导出修复

- 修复自动施肥催熟阶段在作物进入催熟阈值后，因剩余成熟时间进入临界几秒或策略分支额外依赖前置无机肥标记，导致未实际触发催熟的问题；现已改为催熟阶段按 `0..阈值` 持续参与候选判定，并取消 `智能无机，催熟有机 / 种植无机，催熟有机（旧策略）` 对前置无机肥成功记录的硬依赖，确保到达阈值后能正常执行催熟。
- 修复自动施肥旧策略 `种植无机，催熟有机（旧策略）` 在同一块地连续种下相同作物时，可能复用上一轮施肥标记，导致种植后不再自动补无机肥的问题；现已改为结合新一轮成熟时间重新识别种植轮次，并在种植后进入施肥前补一次状态刷新。
- 修复自己农场地块进入 `枯萎 / dead` 状态后，补充逐块收取逻辑仍只识别 `mature`，导致这类地块不会进入后续处理的问题；现已把枯萎地块一并纳入补处理候选。
- 修复日志中心“导出任务日志”在后端返回空文本时直接导出空白文件的问题；现在会优先导出后端持久化日志，若内容为空则自动回退导出页面当前已展示的自动农场日志。

## 2026-04-26

### 自动收获、独立调度与实验性并发

- 自动收获已从基础任务中拆成独立调度任务，并在自动农场页新增独立间隔配置，不再和基础任务共用同一执行节奏。
- 自动种植已拆成独立调度任务，新增独立“种植检测间隔”，调度中心可单独查看和调整其优先级。
- 自动收获默认改为“仅在自己农场页面时执行”，不会为了收获主动把界面从好友农场切回自己农场，降低对当前页面和其他任务的干扰。
- 新增“并发收获（实验性）”开关：开启后，自动收获会尝试在其他自动农场任务执行期间并发触发；该功能默认关闭，并明确标注可能存在状态竞争或任务异常。

### 调度中心与日志细化

- 调度中心新增“后端真实任务调度”视图，不再只管理控制页轮询；自己农场基础任务、自动收获、自动种植、自动施肥、好友偷菜、好友帮忙已接入真实后端任务队列。
- 后端任务面板中的间隔展示文案已明确为“当前生效间隔(秒)”，与自动农场页中的本地草稿值区分开，避免误判为同一来源。
- 自动农场日志已从“轮次级汇总”继续细化为“步骤级实时日志”：种植策略解析、背包优先尝试、施肥阶段开始/结束、好友巡查进入/跳过/完成等步骤现在都会即时输出。
- 自动农场任务日志已接入本地落盘 `data/task-events.ndjson`，日志中心支持导出任务日志文本。

### 消息推送降噪与异常解析

- 异常推送已增加“同类异常 1 小时内不重复推送”的降噪机制。
- 常见异常已改为优先输出业务化文案，不再直接抛原始报错：包括肥料不足、仓库刷新失败、运行时繁忙、连接异常、请求/协议异常等。

### 仓库页与出售总开关

- 仓库页已新增出售总开关；关闭后，后台自动出售与页面手动出售都会被统一阻止。
- 仓库页单项出售按钮已改成更明确的“手动出售”，出售设置、预计出售与后台调度提示会同步受总开关影响。

### 好友黑白名单规则

- 好友农场页新增独立“好友黑白名单规则”区域，集中管理白名单、黑名单及其应用范围。
- 白名单与黑名单都支持分别配置到“偷菜 / 帮助”范围；同一动作范围若被白名单占用，黑名单对应范围会自动让位。
- 好友卡片新增“加入黑名单 / 加入白名单”以及对应移出操作，规则预览区支持直接移出手动规则。
- 原先依赖“蒙面好友”名称的过滤逻辑已去除，改为纯等级阈值过滤：现在只根据好友等级判断是否命中低等级过滤规则，文案也已同步更新。

## 2026-04-24

### 土地详情、作物阶段与资源目录整理

- 土地详情页的作物状态已改为优先按 `成熟倒计时 + grow_phases + 当前季次` 推导，不再单纯依赖运行时 `current_stage`；像“实际已到大叶期但运行时阶段号仍停留在前一阶段”的情况，页面会按推导结果显示。
- 土地详情卡片现在会按当前阶段切换作物图片；阶段图优先按 `phaseName` 命中，只有在缺少阶段名时才回退到阶段序号。
- 土地详情卡片中的阶段文案已调整为更直接的格式：单季显示 `大叶期 4/6`，多季显示 `第1季 · 大叶期 4/6`；单季作物不再额外显示 `季数 1/1`。
- 作物图片资源已统一收敛到 `gameConfig/plant_images/`：`stages/作物` 同时承担主图与阶段图，`default/400.jpg` 作为默认兜底图；旧的 `seed_images_named/`、`categorized_downloads/` 以及项目根目录 `400.jpg` 已清理，不再参与运行链路。
- 现有 `140` 个作物的主图已全部迁移到 `stages/作物` 的 `00_作物图 / 00_主图` 口径，主图显示不再依赖旧的 seed 图目录。

### 仓库图片映射与黄金果实

- 仓库页现已支持读取 `gameConfig/plant_images/stages/` 下各资源目录中的 `*_mapping.json`，按 `item_id` 直接补齐物品名称、等级、稀有度与图片。
- 新接入的 `狗粮`、`化肥道具`、`活动货币`、`礼包与宝箱` 资源包已可在仓库页显示对应图片，不再只依赖 `ItemInfo.json` 原有字段。
- `黄金果实_mapping.json` 已接入仓库资源映射；仓库中出现 `104xxxx` 黄金果实时，会优先显示 `黄金·xxx` 名称与对应黄金主图，不再继续沿用旧的“变异xxx”名称。

### 智能施肥策略与调试日志

- 自动施肥中的 `种植无机，催熟有机` 已升级为“智能无机，催熟有机”：无机肥会按当前季最长生长期命中时机施肥，缺少可靠阶段信号时会回退旧行为。
- 多季作物开启 `多季补肥` 后，第 2 季也会按该季可走阶段重新判断智能无机施肥时机，不再沿用第一季口径。
- 自动农场日志新增智能施肥决策信息：未命中智能施肥阶段时会显式记录“等待智能施肥阶段”，并附带当前季次、当前阶段与目标阶段，方便后续对照小游戏浮层调试。

### 自动施肥效率优化

- 自动农场中的批量施肥已改为默认按同类目标地块整组执行，不再固定按 `2` 块一组切分；在地块较多时，会减少重复开关施肥交互面板带来的额外耗时，更接近自动种植的整批处理方式。
- 自动施肥在真正执行前会重新校验地块状态；空地、枯萎地块和非生长期地块会直接跳过，不再继续轮询施肥。
- 自动施肥现已只保留批量主跑；后续批量补跑与单块兜底已移除，施肥相关 CDP 调用超时提升到 `60s`，避免整批执行在 `8s` 默认超时下被中途截断。
- 批量主跑即便遇到整批超时或部分地块未返回结果，也会在本轮为对应地块补齐失败标记，避免地块施肥记录留空。
- 自动农场日志新增 `自动施肥调度` 汇总，可直接看到本轮批量主跑、批量补跑、单块补跑以及拦截重复补跑的地块数量；当前默认仅保留批量主跑，其余计数通常为 `0`。

### 消息推送与异常识别

- 消息推送的异常通知不再只识别“连接超时累计达到阈值”这一种情况；现已扩展覆盖网络/连接、运行时/上下文、请求/协议、持久化/状态、调度/执行、资源/数据等多类异常。
- 超时异常判断已补齐到项目当前链路里的主要超时来源，包括 `等待自动农场空闲超时`、`AUTO_FARM_RUNTIME_BUSY_TIMEOUT`、`request timeout`、`ETIMEDOUT` 等；这类超时仍按配置阈值累计触发异常通知。
- 自动农场内部 `warn / error` 级事件已接入消息推送异常判断，不再只依赖网关日志文件；调度失败、持久化失败、运行时未就绪等异常现在也会进入指定渠道推送。
- 网关启动时会自动把 `stdout / stderr` 镜像写入 `logs/gateway-out.log` 与 `logs/gateway-err.log`，消息推送的日志监控现在有了稳定输入源，不再出现配置了日志监控但实际无日志可扫的情况。

### 自动种植与背包优先级

- 自动种植策略中的 `背包优先` 已新增二级展开菜单，可直接在控制页里为真实背包种子勾选并排序，自定义背包种子优先级。
- 背包优先级列表改为只展示当前背包里真实存在且有库存的种子；不再把不存在的 `0` 库存种子整表展示出来。
- 新增 `强制保留优先级` 开关：开启后，即便背包暂时没有该种子，已勾选的优先级仍会保留在前端列表中；种子后续重新掉落回背包时，会继续按原优先级优先种植。关闭后刷新背包会自动移除这些无库存优先级。
- 修复背包种子优先级刷新后前端勾选状态丢失的问题：页面刷新、仓库后台刷新或背包列表晚于配置返回时，不会再把已保存优先级误清空。
- 修复自动农场实际执行时未透传背包优先级配置的问题：之前虽然前端与配置文件已保存优先级，但后台执行仍可能按默认背包顺序种植，现已改为真正按保存顺序执行。
- 背包优先策略已改为单轮连续补种：当优先级 1 的种子数量不足以种满全部空地时，会在同一轮内继续尝试优先级 2、优先级 3，而不是等到下一轮。
- 自动农场每轮执行 `背包优先` 时，都会重新读取当前背包种子，不会沿用上一轮旧背包快照。
- 爱心果 `seedId=20046` 已标记为 `不可种植`：即便背包中存在，也只会在列表中显示不可种植状态，自动种植执行时会跳过，不会进入背包优先策略。
- 背包优先级卡片中的库存徽标文案已由 `背包 xxx` 调整为 `数量 xxx`，与其余仓库/数量展示口径保持一致。

### 仓库出售设置体验优化

- 仓库页的出售能力拆分为独立 `出售设置` 区域；仓库浏览筛选与出售配置不再混在同一行里，出售种类会显示在上方，自动出售时间调整到下方，减少误操作。
- 自动出售种类改为固定显示 `果实 / 种子 / 道具 / 材料` 四类真实可售类别；即使当前仓库里暂时没有该类物品，也会继续保留选项，便于提前配置。
- 后台定时出售已接入这组出售种类筛选；保存配置后，后台每轮自动出售只会处理已勾选类别中的可预估出售物品，不再默认扫描所有仓库类别。
- 仓库页新增“按出售范围选中”快捷操作，可按当前出售设置一键选中页面里匹配的可售物品，再配合批量出售直接处理。

## 2026-04-22

### 仓库页与出售链路

- 新增独立 `仓库` 页签，放在 `土地详情` 与 `运行时` 之间；仓库数据改为后台隐藏打开仓库后读取，并把四个仓库页签中的物品聚合为统一预览。
- 仓库预览新增本地缓存；首次进入仓库页或远端临时读取失败时，会优先回退展示最近一次成功读取到的仓库快照，避免页面出现整块空白。
- 仓库调度已接入后台定时刷新：默认每 `5` 分钟自动刷新一次仓库缓存；自动出售默认每 `12` 小时执行一次，若配置为 `0` 则每次刷新仓库时都尝试出售可预估物品。
- 手动出售与自动出售都已接入账户状态“运行统计”；除 `出售` 件数外，新增 `出售估值` 金币统计，并按天写入自动农场统计文件。
- 修复仓库聚合卡片出售时把总数量错误压到单个仓格导致“数量不足”的问题：现在会按同类物品命中多个真实仓格，保留真实 `uid` 后按批量出售链路提交。
- 修复仓库出售匹配与结果观察逻辑：统一仓库展示与卖出请求的物品 ID 解析规则，卖出后改为重新扫描四个仓库页签并聚合比对，减少 `warehouse_sell_no_match` 与 `warehouse_sell_result_not_observed` 误报。
- 优化仓库与自动农场运行时冲突：仓库刷新/出售会等待自动农场当前轮次结束后再执行，不再直接因“自动农场运行中”而失败。
- 优化刷新交互：手动刷新仓库与自动出售已合并为同一次仓库会话，刷新时会先扫四个页签，再按需要直接复用当前已打开的仓库执行自动出售，减少重复开关仓库。

## 2026-04-21

### 账户状态统计持久化

- 账户状态页中的自动农场“今日统计”已改为本地按天持久化，不再只存在内存里；网关或项目重启后会自动恢复当天统计，不会被清空。
- 新增统计目录 `data/account-stats/auto-farm-daily/`，按 `YYYY-MM-DD.json` 保存每日运行统计，便于后续排查与归档。
- 统计会按北京时间（UTC+8）自动隔日切换到新文件，并自动保留最近 `30` 天历史，超出的旧文件会自动清理。
- 自动农场管理器与网关已接入这套每日统计存储：启动时加载最近 30 天记录，运行过程中每轮自动写回当天统计。

### 账户状态页统计增强

- 账户状态页中的“今日统计”卡片升级为“运行统计”，支持直接切换查看 `今天 / 近三天 / 近七天 / 近十五天 / 近三十天` 的汇总数据。
- 新增 `指定日期` 下拉，可单独查看最近 30 天内任意一天的统计明细；切回区间视图时无需刷新页面。
- 统计卡额外补充了查看范围、覆盖天数、总运行轮次和最近更新时间，让多日数据更容易对照。

### 调度中心与串行执行

- 控制页新增独立 `调度中心` 页签，放在“自动农场”旁边，用来统一管理常驻任务的全局执行顺序。
- 新增前端串行调度器配置：支持总开关、任务间缓冲时间、任务启停、优先级、轮询间隔和运行模式，并支持一键上移/下移优先度与恢复推荐值。
- 调度中心会显示当前运行队列、下一次预计执行时间和最近一次结果，用来降低多个页面任务并发轮询导致的小程序卡顿或白屏风险。

### 自动农场细分开关

- 自动农场 `模块开关 -> 基础任务` 现在支持二级动作拆分：`收获 / 除草 / 浇水 / 杀虫` 四项可分别开启或关闭，默认仍全部开启。
- 基础任务二级动作已真正接入执行链，不是仅前端展示；配置会持久化到农场配置，并透传到自动农场执行器。
